Send and receive transactions using the API


Transactions are used to store data on the Tangle and to query data from the Tangle. You can create complex and powerful transactions for many applications. Get started by learning these simple transactions:
  1. Sending and fetching zero-value transactions
  2. Sending and fetching value transactions used to deposit or withdraw IOTA tokens
  3. Sending and fetching data transactions used to send data
Lesson 2 teaches how to send a transaction using Masked Authenticated Messaging (MAM).

Sending and fetching zero-value transactions
Learning zero-value transactions provides a foundation for learning value transactions, data transactions and messages. They are called zero-value because the value field contains 0 (zero) IOTA tokens.

Sending and fetching value transactions
Value transactions are used to send and receive IOTA tokens. The value field contains the number of IOTA tokens to be sent. Sending tokens requires you to acquire some IOTA tokens and store them in an address. Fortunately, the DevNet Faucet "drips" IOTA test tokens that you can use for practice. While these test tokens function like currency, they cannot be exchanged for other cryptocurrency or fiat currency. Once you have tokens, you must keep track of your balance. Sending and fetching data transactions
Data transactions are zero-value transactions that have data stored in the tag or message fields. Data stored in the message field must be converted to trytes. The tryte alphabet is "A-Z9". ZMQ is a tool that can be used to query data transactions stored on the Tangle.
